# Логика обработки одного аудио (10 шагов)

| Шаг | Название | Модуль |
|-----|----------|--------|
| 1 | Загрузка аудио + `processing_job` | `routers/api.py` |
| 2 | Предобработка (mono 16kHz, тишина, метаданные) | `services/preprocessing.py` |
| 3 | ASR (текст, сегменты, таймкоды, confidence) | `services/asr/` |
| 4 | Сегментация логических блоков | `services/segmentation.py` |
| 5 | LLM-разбор JSON (OpenAI/Claude → strict JSON) | `services/llm/` |
| 6 | Нормализация значений | `services/normalizer.py` |
| 7 | Валидация | `services/validator.py` |
| 8 | Сохранение в БД | `services/processing.py` |
| 9 | Отображение (long/wide, предупреждения) | Frontend `App.tsx` |
| 10 | Экспорт Excel | `services/excel_export.py` |

Оркестратор: `services/processing.py` — `run_session_processing()`.

При загрузке создаётся `ProcessingJob` со статусом `pending`, шаг 1.
Кнопка «Обработать» запускает шаги 2–9 (или Celery-очередь).

## Маркеры сегментации (шаг 4)

- Паузы > 1.5 сек (ASR-сегменты)
- «далее», «следующая запись», «следующий перегон»
- Смена контекста: перегон, км, пикет, путь

## Нормализация (шаг 6)

| Вход | Выход |
|------|-------|
| путь второй | 2 |
| пикет 5 плюс 20 | 5+20 |
| миллиметров | мм |
| километров в час | км/ч |

## Валидация (шаг 7)

- Формат км: `\d+(\.\d+)?`
- Формат пикета: `\d+(\+\d+)?`
- Допустимые единицы: мм, см, м, км/ч, °, ‰
- Диапазоны значений для износ/просадка/уровень
- Обязательные поля: км/пикет/перегон или дефект

Ошибки → `validation_errors` на записи, предупреждения → `validation_warnings` на сессии.
